

## Problem: M-Pesa B2C Not Configured

The withdrawal failed because two required secrets are missing for M-Pesa B2C (Business-to-Customer) payouts:

- `MPESA_B2C_INITIATOR_NAME` -- not set
- `MPESA_B2C_SECURITY_CREDENTIAL` -- not set

The shortcode (`MPESA_SHORTCODE`) is already configured and working.

### What These Secrets Are

1. **MPESA_B2C_INITIATOR_NAME**: The API operator username configured on the Safaricom Daraja portal for your shortcode (e.g., "testapi" for sandbox, or a custom name for production).

2. **MPESA_B2C_SECURITY_CREDENTIAL**: The encrypted password for the initiator. This is generated by encrypting the initiator password with Safaricom's production certificate. You generate it on the Daraja portal under "APIs > B2C" or by using their certificate tool.

### How to Get These Credentials

1. Log into [Safaricom Daraja Portal](https://developer.safaricom.co.ke/)
2. Go to your app for shortcode 4015351
3. Under B2C API settings, find or set your **Initiator Name** (the API operator username)
4. Generate the **Security Credential** by encrypting your initiator password with the production certificate (there is a tool on the portal under "APIs > B2C > Generate Security Credential")

### Technical Details

The edge function `mpesa-b2c-payout/index.ts` checks for these at lines 110-111:
```typescript
const initiatorName = Deno.env.get('MPESA_B2C_INITIATOR_NAME');
const securityCredential = Deno.env.get('MPESA_B2C_SECURITY_CREDENTIAL');
```

When both are missing, it falls through to line 122 and returns the "M-Pesa B2C not configured" error, marking the withdrawal as `failed`.
